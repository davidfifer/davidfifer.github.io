CSV Data Cleaner

Overview

CSV Data Cleaner is a robust Python utility designed to sanitize, normalize, and validate CSV datasets. It automates common data‑cleaning tasks such as trimming whitespace, removing empty rows, fixing malformed values, enforcing schema rules, and generating detailed reports. The tool emphasizes reliability, transparency, and configurability, making it ideal for analytics pipelines, ETL workflows, and automated preprocessing.

Architecture & Design

The project follows a modular architecture that separates parsing, validation, transformation, reporting, and configuration into dedicated components. This structure makes the tool easy to extend with new cleaning rules or custom validation logic.

  • Parser Module: Loads CSV files, detects delimiters, and normalizes input.
  • Validator: Applies schema rules, checks required fields, and flags invalid rows.
  • Transformer: Cleans and standardizes values (trimming, type coercion, normalization).
  • Reporter: Generates structured summaries of issues, fixes, and statistics.
  • Config Loader: Reads config.json to define cleaning rules and schema constraints.

Key Features

  • Whitespace Normalization: Trims leading/trailing spaces across all fields.
  • Empty Row Removal: Automatically filters out blank or incomplete rows.
  • Type Coercion: Converts numeric and boolean fields to consistent formats.
  • Schema Validation: Ensures required columns exist and values match expected types.
  • Duplicate Detection: Identifies and optionally removes duplicate rows.
  • Custom Rules: User‑defined transformations via config.json.
  • Detailed Reporting: Summaries of errors, corrections, and dataset statistics.
  • Dry‑run Mode: Preview all changes without modifying the original file.
  • Safe Output Handling: Writes cleaned data to a new file to prevent accidental overwrites.

Usage Examples

Basic Cleaning

python csv_data_cleaner.py --input "data.csv" --output "cleaned.csv"

Dry‑run Mode

python csv_data_cleaner.py --input "raw.csv" --dry_run

Custom Schema Validation

python csv_data_cleaner.py --input "users.csv" --config "schema_config.json"

Verbose Logging

python csv_data_cleaner.py --input "sales.csv" --verbose

Configuration

The tool reads a config.json file that defines schema rules, cleaning behaviors, and custom transformations. This allows users to tailor the cleaner to their dataset without modifying code. 

{
  "dry_run": false,
  "required_columns": ["id", "name", "email"],
  "type_map": {
    "id": "int",
    "email": "string",
    "active": "bool"
  },
  "remove_duplicates": true,
  "custom_rules": {
    "email": "lowercase",
    "name": "trim"
  }
}

Engineering Challenges & Solutions

  • Malformed Rows: Implemented robust parsing with fallback strategies.
  • Inconsistent Types: Added type coercion and configurable type maps.
  • Large Datasets: Stream‑based processing to reduce memory usage.
  • Schema Drift: Configurable validation rules to detect missing or unexpected columns.

Future Enhancements

  • Interactive web UI for rule configuration.
  • Support for JSON, TSV, and Parquet formats.
  • Plugin system for advanced transformations.
  • Integration with ETL orchestration tools.
