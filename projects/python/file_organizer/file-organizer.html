File Organizer

Overview

File Organizer is a cross‑platform Python utility designed to bring structure to cluttered directories. It scans a target folder, classifies files by extension or last‑modified date, and moves them into clean, predictable subfolders. The tool emphasizes safety, configurability, and operational transparency through dry‑run mode, verbose logging, and resilient file‑locking detection.

Architecture & Design

The project follows a modular architecture that separates scanning, classification, movement, logging, and configuration into distinct components. This makes the tool easy to extend (e.g., adding new sort modes or cloud‑sync behavior).

  • Scanner Module: Walks the directory tree and collects file metadata.
  • Classifier: Determines destination folders based on extension or timestamp.
  • Executor: Moves files, respecting dry‑run mode and retry logic.
  • Logger: Provides structured logs with adjustable verbosity.
  • Config Loader: Reads config.json to customize behavior.

This separation of concerns ensures the tool remains maintainable and testable as it grows.

Key Features

  • Sort by Extension: Groups files into subfolders based on file type.
  • Sort by Modified Date: Organizes files into YYYY‑MM‑DD folders.
  • Dry‑run Mode: Shows all planned actions without touching the filesystem.
  • Verbose Logging: Multiple log levels for debugging or quiet operation.
  • Config‑Driven Behavior: File‑type mappings and flags defined in config.json.
  • Automatic Directory Creation: Creates destination folders on demand.
  • Cross‑Platform File‑Lock Detection: Uses msvcrt (Windows) and fcntl (Unix).
  • Retry Logic: Retries locked or unavailable files with exponential backoff.
  • Graceful Error Handling: Logs warnings without interrupting the process.

Usage Examples

Basic Sorting

python file_organizer.py --folder_path "C:/Downloads"

Sort by Modified Date

python file_organizer.py --folder_path "./data" --sort_mode date

Dry‑run Mode

python file_organizer.py --folder_path "./incoming" --dry_run

Verbose Logging

python file_organizer.py --folder_path "./docs" --verbose

Configuration

The tool reads a config.json file that defines file‑type mappings, dry‑run defaults, and retry behavior. This allows users to tailor the organizer to their workflow without modifying code. 

{
  "dry_run": false,
  "retry_delay": 1.0,
  "max_retries": 3,
  "extension_map": {
    "images": ["jpg", "png", "gif"],
    "documents": ["pdf", "docx", "txt"]
  }
}

Engineering Challenges & Solutions

  • Locked Files: Implemented OS‑specific lock detection and retry logic.
  • Cross‑Platform Paths: Standardized path handling using pathlib.
  • Large Directories: Optimized scanning to avoid unnecessary I/O.
  • Safety: Added dry‑run mode and verbose logs to prevent accidental moves.

Future Enhancements

  • GUI wrapper using Tkinter or PyQt.
  • Cloud‑sync support (OneDrive, Google Drive).
  • Plugin system for custom sort modes.
  • Parallel file movement for large datasets.
